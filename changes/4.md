## AI integration approach

**Chosen:** MCP server — app exposes project tools via the Model Context Protocol.
Users connect their own AI client (Claude Desktop, Cursor, etc.) using a personal token.
The model runs in the user's environment under their own subscription — the app never handles API keys.

**Rejected:** "Bring your own API key" chat widget (R3.4). Removed in R4.1 — free API tiers are
exhausted immediately, billing is separate from site usage, and the UX friction was high.

### R4.2 — MCP server ✓
`POST /api/mcp` — implements MCP protocol (JSON-RPC 2.0, protocol version `2024-11-05`).

**Auth:** `Authorization: Bearer <mcpToken>` header → `User.mcpToken` DB lookup. Returns 401 if
token missing, invalid, or user not ACTIVE.

**Methods handled:**
- `initialize` → server capabilities
- `ping` → empty result
- `notifications/initialized` → 204
- `tools/list` → list of 4 tools
- `tools/call` → dispatch to tool executor

**Tools:**
| Name | Description | Args | RBAC |
|------|-------------|------|------|
| `list_projects` | All projects user can see | none | VIEW_ALL_PROJECTS or assignment |
| `get_project_detail` | Deliverables + open action items | `projectId` | assignment or VIEW_ALL_PROJECTS |
| `create_action_item` | Create action item on a project | `projectId, description, deadline?` | ASSIGN_ACTION_ITEMS or LEAD/SUBLEAD |
| `create_status_update` | Post a weekly status update | `projectId, meetingDate, plannedWork, actualProgress, blockers, nextWeekGoals` | POST_MEETING_TRACKING or SUBMIT_STATUS_UPDATES or LEAD/SUBLEAD |

All tools return JSON text in `content[0].text`.

## MCP client setup (for end users)

Paste into `claude_desktop_config.json` or Cursor MCP settings:

```json
{
  "mcpServers": {
    "seed-tracker": {
      "url": "http://localhost:3000/api/mcp",
      "headers": {
        "Authorization": "Bearer <your-token>"
      }
    }
  }
}
```

Replace `localhost:3000` with the deployed URL when live.
