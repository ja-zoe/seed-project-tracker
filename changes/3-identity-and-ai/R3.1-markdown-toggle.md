# R3.1 — Plain-text / Markdown toggle on description fields

**Status:** done
**Files:** `src/components/markdown-editor.tsx` (+ every page that renders `MarkdownEditor`)

## Spec

**Problem:** `MarkdownEditor` was the only input mode for descriptions (deliverables, subtasks, status updates). Some users want plain text without markdown syntax.

**Approach:**
- Add a small `MD / Plain` pill toggle above the textarea (top-right of the label row).
- `MarkdownEditor` gains an optional `allowToggle` prop (default `true`).
- Toggle is local component state — no DB storage. The stored value is always the raw text; mode only affects presentation.
- MD mode: monospace textarea, markdown placeholder hints, Write/Preview tabs (react-markdown + remark-gfm, already installed).
- Plain mode: proportional font textarea, simplified placeholder, no hints.

**No DB changes.**

## Notes / log

- 2026-06-26 — Added `allowToggle` prop and `mode` state (`"md" | "plain"`) to `markdown-editor.tsx`. Plain mode: proportional font, no Write/Preview tabs, simplified placeholder. Switching modes never clears the value. Defaults to MD, no persistence (per decision).
