# R3.2 — Inline subtask delete with slide-in confirmation

**Status:** done
**Files:** `src/components/sortable-deliverables.tsx`, `src/lib/actions/deliverables.ts`

## Spec

**Problem:** `SortableDeliverables` showed an "Edit" link per subtask but no delete capability.

**Approach (UX):** Add an `XCircle` (Phosphor, Bold, muted) trigger right of "Edit". On click, a confirmation micro-interaction: Edit + X slide out left (`translateX` + opacity, ~200ms ease-out) while "Yes / No" slide in from the right. "Yes" in `text-[#A4503C]` (clay), "No" muted. "No" reverses; "Yes" calls `deleteSubtask` and fades the row out. State: one `confirmingDelete: string | null` in `SortableDeliverables`. CSS transitions only (Tailwind `transition-all duration-200`), no GSAP.

**New server action:** `deleteSubtask(subtaskId: string)` in `src/lib/actions/deliverables.ts` — auth: project membership (any role) or `MANAGE_MILESTONES`; deletes + revalidates project path.

**No DB changes.**
