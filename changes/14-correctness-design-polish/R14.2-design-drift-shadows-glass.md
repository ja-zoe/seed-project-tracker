# R14.2 — Remove heavy shadows & glassmorphism (design-system drift)

**Status:** tests passing
**Files:** `src/components/ui/dialog.tsx`, `src/components/semester-calendar.tsx`,
`src/components/sortable-deliverables.tsx` (popover/menu className strings)

## Spec

**Problem:** CLAUDE.md / CONTEXT.md state the Forest Floor system has **"No heavy shadows, no gradients, no
glassmorphism."** The codebase drifted:
- `ui/dialog.tsx` overlay uses `supports-backdrop-filter:backdrop-blur-xs` → **glassmorphism**.
- `semester-calendar.tsx` event modal uses `shadow-lg` → **heavy shadow**.
- `sortable-deliverables.tsx` menus/popovers use `shadow-md` at 5 sites → **heavy shadows**.

The established elevation idiom elsewhere is **flat: `bg-card border border-border rounded-*`** (cards,
panels, the action-item modal). Floating layers should read as a crisp bordered surface, not a blurred,
shadowed one.

**Approach (route through the bridged design / design-system skills to confirm the in-system treatment):**
- Dialog overlay (`ui/dialog.tsx:34`): drop `supports-backdrop-filter:backdrop-blur-xs`; keep the dim
  scrim (`bg-black/10`). No blur.
- Floating surfaces (modals, menus, pickers): replace `shadow-lg`/`shadow-md` with the flat idiom —
  `border border-border` (most already have it) plus, at most, a **very subtle** `shadow-xs` for
  separation if the design skill deems a hairline shadow acceptable; otherwise border-only. Decide one
  consistent treatment and apply it to all six sites so popovers match.
- Keep geometry/spacing/radius unchanged; this is purely the shadow/blur layer.

**Decision (resolved):** use **border-only elevation** (drop the blur, replace `shadow-md/lg` with no shadow
or `shadow-xs` max), matching the flat cards already in the system — the most architecture-consistent
reading of "no heavy shadows." Record the exact final class in Notes.
