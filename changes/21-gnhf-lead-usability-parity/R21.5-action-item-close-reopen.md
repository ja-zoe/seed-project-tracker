# R21.5 — Action-item close/reopen required a global permission leads may lack

**Status:** done (merged in `d6110a9`)
**Files:** `src/lib/actions/action-items.ts`, `src/app/(app)/projects/[id]/page.tsx`, `src/app/(app)/action-items/page.tsx`

## Spec

**Problem:** `closeActionItem`/`reopenActionItem` required the global CLOSE_ACTION_ITEMS permission, while
create/update/delete keyed on project membership — so a lead could *delete* an item on their own project but
not mark it done or reopen it. The seeded Project Lead role masks this (it includes CLOSE_ACTION_ITEMS); the
drift surfaces for custom global roles.

**Approach:** Grant LEAD/SUBLEAD via membership in both actions, preserving `closeActionItem`'s existing
owner fast-path (any member may close an item they own — `reopenActionItem` deliberately has no such path,
so its membership check stands alone). Project page passes `canClose = canCloseActionItems || isLeadHere`
to `ActionItemsSection`; the hub computes per-item `canCloseItem(projectId) = canCloseAll ||
leadProjectIds.has(projectId)`, mirroring R21.4's `canEditItem`.

## Tests

Retroactive — verified during the run: typecheck + lead close/reopen on own project via both surfaces. No
dedicated Playwright spec.
