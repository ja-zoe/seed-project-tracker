# R21.7 — MCP status updates were orphans that never cleared the web pending affordance

**Status:** done (merged in `62a5dde`)
**Files:** `src/app/api/mcp/route.ts`

## Spec

**Problem:** Status updates are keyed to lead meetings: the web pending-state (`getPendingLeadMeetings`,
`status/new`) requires a `calendarEventId`-linked StatusUpdate, while the MISSING_SUBMISSION notification
keys only on `submittedAt` recency. MCP `create_status_update` created an unlinked orphan with a
caller-supplied `meetingDate` and `isLate` hardcoded false — silencing the notification but leaving the web
"Submit Project Standing" affordance stuck pending, causing persistent nagging and duplicate submissions.

**Approach:** The tool now submits against a pending lead meeting (soonest, or a named `calendarEventId`)
via `getPendingLeadMeetings`, deriving `calendarEventId`/`meetingDate`/`isLate`. It returns a clear error
when no lead meeting is open for submission. Added optional `needsHelp`/`helpNeeded` for parity with the web
submit form; removed the now-derived `meetingDate` parameter from the schema.

## Tests

Retroactive — verified during the run: typecheck + MCP submission clears the web pending affordance. No
dedicated Playwright spec.
