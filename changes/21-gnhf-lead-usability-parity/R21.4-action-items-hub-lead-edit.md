# R21.4 — Global Action Items hub blocked lead editing

**Status:** done (merged in `94c2100`)
**Files:** `src/app/(app)/action-items/page.tsx`

## Spec

**Problem:** A lead could edit action items on the project detail page, but the sidebar-linked global
`/action-items` hub gated all editing on the PM-only ASSIGN_ACTION_ITEMS global permission — silently
stripping an ability the backend (`updateActionItem`) and the project page both allow.

**Approach:** Compute per-item editability: `canEditItem = canAssign || leadProjectIds.has(item.projectId)`,
where `leadProjectIds` comes from the user's LEAD/SUBLEAD assignments. Broaden the assignees query (it
previously ran only when `canAssign`) to cover the lead's editable projects so the edit modal has owner
options.

## Tests

Retroactive — verified during the run: typecheck + lead sees edit modal with owner options on own-project
items in the hub. No dedicated Playwright spec.
