# R21.3 — Dead EDIT_OWN_PROJECT permission wired up

**Status:** done (merged in `a8b0563`)
**Files:** `src/lib/actions/projects.ts`, `src/app/(app)/projects/[id]/page.tsx`, `src/components/project-modal.tsx`

## Spec

**Problem:** `EDIT_OWN_PROJECT` was a fully dead permission: granted to the seeded Project Lead role and
rendered in the PM role-editor UI, but checked by no server action or page. `updateProject` required
PM-only MANAGE_PROJECTS, so a lead couldn't edit their own project — including adding the corrective action
plan a BEHIND project requires (the flow R21.1's banner drives).

**Approach:** New `canEditProject()` helper in `projects.ts`: MANAGE_PROJECTS, or EDIT_OWN_PROJECT +
LEAD/SUBLEAD membership. `updateProject` authorizes through it. The project detail page exposes the
Edit-project button and the BEHIND-banner "Add one" trigger via `canEditThisProject`. `ProjectModal` gained
a `canDelete` prop (default true) so leads don't see the PM-only Delete-project button — `deleteProject`
stays MANAGE_PROJECTS-only.

## Tests

Retroactive — verified during the run: typecheck + lead can edit own project / cannot delete. No dedicated
Playwright spec.
