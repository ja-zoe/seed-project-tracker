# R21.12 — Subtask.startDate was a dead column no surface could write

**Status:** done (merged in `8b8d5a3`)
**Files:** `src/components/subtask-modal.tsx`, `src/lib/actions/deliverables.ts`, `src/app/(app)/projects/[id]/deliverables/[did]/subtasks/[sid]/edit/page.tsx`, `src/app/api/mcp/route.ts`, `src/components/sortable-deliverables.tsx`, `src/app/(app)/projects/[id]/page.tsx`

## Spec

**Problem:** `Subtask.startDate` drove the timeline Gantt duration bars (`hasSubBar`) and the Excel export,
but no create/edit form, server action, or MCP tool ever wrote it — an entire column consumed by two read
surfaces and written by zero. (Deliverables had startDate on every form; subtasks only in the schema. The
due-date input even set `min={deliverableStart}`, hinting a range was always intended.)

**Approach:** Start date field on all four write surfaces:
- `SubtaskModal` (create + edit): Start date input with bounds/validation (valid date, within the
  deliverable window, not after due); layout restructured — Assignee full-width above a Start/Due row.
- `createSubtask`/`updateSubtask` persist it; `assertDueWithinDeliverable` renamed to
  `assertSubtaskDatesWithinDeliverable` enforcing start<=due and both within the deliverable window, also
  applied to the inline `updateSubtaskDueDate` path. `updateSubtask` writes startDate only when submitted
  (`formData.has`) so partial callers don't clear it.
- Standalone `/subtasks/[sid]/edit` page: Start Date input for parity.
- MCP `create_subtask`/`update_subtask`: optional `startDate` param + return field, matching the
  deliverable tools.
Threaded `startDate` through the `sortable-deliverables` Subtask type, the project page's subtask mapping,
and the edit-modal prop.

## Tests

Retroactive — verified during the run: typecheck + start dates settable from all four surfaces and rendered
as Gantt duration bars. No dedicated Playwright spec.
