# R21.14 — Deliverable priority not settable at creation

**Status:** done (merged in `b29fb4f`)
**Files:** `src/app/(app)/projects/[id]/deliverables/new/page.tsx`, `src/lib/actions/deliverables.ts`, `src/app/api/mcp/route.ts`

## Spec

**Problem:** Priority drives the default deliverable sort (priority DESC, then orderIndex) and a colored
badge, and was editable via the DeliverableModal and the inline chip — but neither the create page,
`createDeliverable`, nor the MCP create tool accepted it, forcing every new deliverable to Medium and
requiring an immediate follow-up edit.

**Approach:** Priority select (High/Medium/Low, default Medium) on the "Add Deliverable" page beside Group;
`createDeliverable` reads and persists it, validated against the Prisma `Priority` enum. MCP
`create_deliverable`/`update_deliverable` accept an optional `priority` param (schema + executor) and return
it, completing cross-surface parity.

## Tests

Retroactive — verified during the run: typecheck + priority set at creation via web and MCP. No dedicated
Playwright spec.
