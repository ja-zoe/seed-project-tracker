# R21.10 — MCP delete_deliverable still PM-only

**Status:** done (merged in `45cd323`)
**Files:** `src/app/api/mcp/route.ts`

## Spec

**Problem:** After R21.2 aligned the web, the MCP surface had an inconsistency *internal to itself*:
`create_deliverable`, `update_deliverable`, and `delete_subtask` allowed project LEAD/SUBLEAD, but
`delete_deliverable` alone remained MANAGE_MILESTONES-only — a lead could delete a deliverable on the web
but not via the AI assistant.

**Approach:** `delete_deliverable` authorizes LEAD/SUBLEAD via `getProjectMembership`
(MANAGE_MILESTONES || LEAD || SUBLEAD), matching the web `deleteDeliverable` action and its sibling MCP
tools; tool description updated to state the requirement.

## Tests

Retroactive — verified during the run: typecheck + lead-token tool call succeeds. No dedicated Playwright
spec (API surface).
