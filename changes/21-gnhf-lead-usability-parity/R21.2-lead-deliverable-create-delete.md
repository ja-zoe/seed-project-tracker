# R21.2 — Leads couldn't create/delete deliverables on their own project

**Status:** done (merged in `1f17678`)
**Files:** `src/lib/actions/deliverables.ts`, `src/app/(app)/projects/[id]/deliverables/new/page.tsx`, `src/app/(app)/projects/[id]/page.tsx`

## Spec

**Problem:** `createDeliverable` and `deleteDeliverable` required the PM-only MANAGE_MILESTONES permission,
while `updateDeliverable` and all subtask actions already allowed project LEAD/SUBLEAD. A lead on a fresh
project was fully dead-ended: able to edit deliverables that didn't exist and unable to add any.

**Approach:** Allow LEAD/SUBLEAD via `getProjectMembership` in `createDeliverable`/`deleteDeliverable`
(matching `updateDeliverable`'s gate); relax the `/deliverables/new` page guard the same way; pass
`canManage = canManageMilestones || isLeadHere` to `SortableDeliverables` so leads see the Add/Edit/Delete/
reorder affordances.

## Tests

Retroactive — verified during the run: typecheck + lead-role behavioral check on the deliverables surfaces.
No dedicated Playwright spec.
