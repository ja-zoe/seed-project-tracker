# R21.1 — Behind-banner "Add one" corrective-action-plan link 404'd

**Status:** done (merged in `3d70946`)
**Files:** `src/app/(app)/projects/[id]/page.tsx`

## Spec

**Problem:** The BEHIND-project banner's "Add one" corrective-action-plan prompt linked to
`/projects/[id]/edit` — a route that was never built (project editing is done entirely via the
`ProjectModal` component). Every click 404'd, dead-ending the exact escalation flow the banner exists for.

**Approach:** Wire the "Add one" affordance to open the existing `ProjectModal` edit dialog instead of the
dead link. Gate it on `canManage` (MANAGE_PROJECTS) to match `updateProject`'s actual backend permission at
the time (later broadened by R21.3). Extract a shared `editableProject` object reused by both the header
Edit-project button and the banner trigger; remove the unused `canEditProject` variable.

## Tests

Retroactive — verified during the run: `tsc --noEmit` clean (except the pre-existing
`e2e/r14-design-shots.spec.ts` error) and the banner drives the working modal. No dedicated Playwright spec.
