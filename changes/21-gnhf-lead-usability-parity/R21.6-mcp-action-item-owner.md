# R21.6 — MCP tools couldn't assign action-item owners

**Status:** done (merged in `833e527`)
**Files:** `src/app/api/mcp/route.ts`

## Spec

**Problem:** The web owner dropdown lets a lead delegate an action item to any teammate, but the MCP
`create_action_item` tool always self-assigned the caller as owner with no override, and
`update_action_item` couldn't change the owner at all — an input-field parity gap, not a permission gap.

**Approach:** Optional `ownerId` parameter on both tools (schema + executor). `create_action_item` still
defaults to the caller when omitted (backward compatible) and clears the owner on empty string;
`update_action_item` reassigns or clears likewise. Descriptions and return `select`s updated to reflect
owner assignment.

## Tests

Retroactive — verified during the run: typecheck + tool calls with/without `ownerId`. No dedicated
Playwright spec (API surface).
