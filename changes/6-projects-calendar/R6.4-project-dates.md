# R6.4 — Project start & end dates

**Status:** in progress
**Files:** `prisma/schema.prisma` + DB DDL, `src/lib/actions/projects.ts`,
`src/app/(app)/projects/[id]/page.tsx` (header display), `src/lib/utils.ts` (duration helper),
`src/app/api/mcp/route.ts` (`get_project_detail`)

**Shares files with R6.2** (projects.ts, the edit page, MCP project tools). Implement this
feature's **schema first**, then R6.2's edit form/MCP tools consume the new fields. One agent
owns both R6.2 + R6.4.

## Spec

**Problem:** A project has no notion of when it started or ends. New members can't tell how
long a project has been running. We want a start date and an optional end date; when the end
date is absent the project is treated as ongoing ("Present").

**Approach:**

1. **Schema.** Add to `model Project`:
   - `startDate DateTime?`
   - `endDate   DateTime?`
   Both nullable so existing rows don't need backfill. DDL:
   ```sql
   ALTER TABLE "Project" ADD COLUMN IF NOT EXISTS "startDate" TIMESTAMP(3);
   ALTER TABLE "Project" ADD COLUMN IF NOT EXISTS "endDate"   TIMESTAMP(3);
   ```
   Apply via `tsx scripts/apply-schema.ts`, then update `schema.prisma`, `pnpm prisma
   generate`, restart dev server.

2. **Display semantics** (a shared helper in `src/lib/utils.ts`):
   - Effective start = `startDate ?? createdAt` (so legacy projects still show a sensible
     "running for" value).
   - End label = `endDate ? formatDate(endDate) : "Present"`.
   - `projectDuration(start, end)` → human string like `"3 mo"` / `"5 wk"` / `"12 days"`
     measured from effective start to `endDate ?? now`. Keep it compact for the header.

3. **Project header** (`page.tsx`): under the project name/semester line, add a mono-styled
   line: `<startDate> – <Present|endDate> · running 3 mo`. Use the `Calendar` Phosphor icon.

4. **Edit/create forms (via R6.2):** the edit page and `createProject` already receive
   `FormData`; add `startDate`/`endDate` (`<input type="date">`, end optional) and parse them
   in `createProject` + `updateProject` (`raw ? new Date(raw) : null`). This is the field
   plumbing R6.2's edit page renders.

5. **MCP:** include `startDate`/`endDate` in `get_project_detail`'s project payload, and accept
   them in `create_project`/`update_project` (those tools are defined in R6.2 — coordinate so
   the args land in the same edit).

**Edge cases:**
- `endDate` before `startDate`: validate in the form actions and reject with a clear error.
- A project with `endDate` in the past still renders its real end label, not "Present".
