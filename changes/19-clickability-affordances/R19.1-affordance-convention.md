**Status:** planned ‚Äî confirm Open Question 1 (hover palette) in `_set.md`.

## Spec

**Problem:** There's no shared, named convention for "this element is clickable." Hover treatments are
ad-hoc (`hover:text-foreground` here, a tooltip there, a cursor change somewhere else), so the rollout in
R19.2 would otherwise reinvent the cue per component and drift. Define the cue once.

**Approach:** Add small utility classes in `globals.css` (Tailwind v4 ‚Äî `@layer` utilities or plain CSS
classes bridged via the existing `@theme inline` tokens; no config file). Built on Forest Floor tokens, no
new colors, no underlines/shadows/gradients (CONTEXT style rules).

- **`.clickable`** ‚Äî for interactive text/labels/rows: `cursor: pointer;` + a `color` transition to the
  foreground forest (`#2E4034`) on hover (`transition-colors`, ~150ms). For elements that are already
  `foreground`, the cue can be a subtle opacity/background tint instead ‚Äî define a `.clickable` that works
  on muted text (muted-foreground ‚Üí foreground) as the common case.
- **`.clickable-icon`** ‚Äî for icon buttons: always `cursor: pointer;` + a `color` shift on hover
  (muted-foreground ‚Üí foreground), matching the icon-button idiom already used on the account page.
- **`.clickable-danger`** ‚Äî variant whose hover color is clay `#A4503C` (delete/remove triggers), matching
  the existing convention so destructive affordances read consistently.

Keep them composable with existing Tailwind utilities (so a component can do
`className="clickable-icon"` or add `hover:` overrides). Document the rule in CONTEXT.md "Standing
decisions": *every interactive element must carry a hover cue; clickable icons always get pointer cursor +
color shift; use `.clickable`/`.clickable-icon`/`.clickable-danger`; do not signal interactivity on static
display elements.*

**State:** none ‚Äî pure CSS + docs. **Constraints:** respect `prefers-reduced-motion` (the color transition
is fine; avoid motion-heavy cues). Don't introduce a focus-only style that removes existing focus rings ‚Äî
keep keyboard focus states intact.
