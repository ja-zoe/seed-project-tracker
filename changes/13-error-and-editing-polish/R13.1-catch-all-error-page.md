# R13.1 — Catch-all error & not-found pages

**Status:** planned
**Files (new):** `src/app/global-error.tsx`, `src/app/(app)/error.tsx`, `src/app/not-found.tsx`
(optional: `src/app/(app)/not-found.tsx`)

## Spec

**Problem:** There is no catch-all unhandled-error UI. When a Server Component or action throws, the user
sees Next's raw error overlay (dev) or a blank/unstyled default (prod). The app already calls `notFound()`
in many routes but has no styled 404 either. We want a branded, friendly fallback for both.

**Approach:** Use the App Router's error/not-found conventions, styled in the Forest Floor system
(Instrument Serif heading, mono labels, Phosphor icons, `#A4503C` clay accent for the error case — no new
colors).

- **`src/app/(app)/error.tsx`** (Client Component — error boundaries must be `"use client"`): catches
  errors thrown anywhere under the authenticated app shell. Because it renders *inside* `(app)/layout.tsx`,
  the sidebar/session chrome stays; only the page body is replaced. Props `{ error: Error & { digest?: string }, reset: () => void }`.
  Show a "Something went wrong" card with a short message, a **"Try again"** button (`onClick={reset}`),
  and a link to `/dashboard`. `useEffect(() => console.error(error), [error])` for visibility. Do **not**
  render `error.message` verbatim in prod (Next already strips it server-side; show a generic line).
- **`src/app/global-error.tsx`** (Client Component): the last-resort boundary that catches errors in the
  **root layout** itself. It must render its own `<html><body>` (it replaces the root layout). Wire the
  font CSS variables (`--font-sans`, `--font-display`, `--font-mono`) the same way `src/app/layout.tsx`
  does, set the canvas background, and show the same "Something went wrong" + "Try again" (`reset`) UI.
  This is intentionally minimal/standalone since the normal providers aren't available here.
- **`src/app/not-found.tsx`** (Server Component): global 404 for `notFound()` calls and unknown routes —
  a "Page not found" card with a link back to `/dashboard`. (Optionally also `src/app/(app)/not-found.tsx`
  so in-shell 404s keep the sidebar; recommend adding it for the authenticated 404s the app actually
  triggers.)

**Constraints/notes:** error boundaries only catch errors in their **subtree during render** — they do
not catch errors in event handlers or already-sent responses; that's fine for this goal. Keep these
components dependency-light (no GSAP/providers in `global-error`). No data fetching.
