## Spec

**Problem:** A user can only set their name/nickname during the one-time `/profile/setup` flow (which lives
outside the app shell and `redirect`s to `/dashboard`). There is no in-app place for an ordinary user to
edit their **nickname** or correct their name later. The sidebar's "Account" → `/account` page only does
MCP token generation. Everyone needs a settings page.

**Approach:** Add a **Profile** section to the existing `/account` page (already linked for all users in the
sidebar), above the current "AI Integrations" (MCP) section; retitle the page heading to **"Account"**.

- *Server action.* In `src/lib/actions/profile.ts` add `saveProfile(formData)` — same validation/update as
  `updateProfile` (firstName + lastName required, nickname optional, recompute `name = "first last"`), but
  **`revalidatePath("/account")` instead of `redirect("/dashboard")`** so the user stays on the page. Keep
  `updateProfile` as-is for the setup flow. (Decision: a separate action, not a flag, so the setup flow's
  redirect behaviour is untouched.)
- *Page.* `account/page.tsx` (Server Component) loads the user's `firstName`, `lastName`, `nickname`,
  `email` and renders `<ProfileSettingsForm … />` above `<MpcTokenSection … />`. Email is shown
  **read-only** (it's the CAS identity, not user-editable). Page heading → "Account"; keep the existing
  eyebrow.
- *Client form.* `src/components/profile-settings-form.tsx` — controlled inputs for firstName / lastName /
  nickname seeded from props, a `useTransition` submit calling `saveProfile`, a disabled+"Saving…" pending
  state and a brief "Saved" confirmation. Match the Forest Floor form styling used elsewhere (mono labels,
  `bg-card border border-border`, Phosphor icons). `data-testid`s on the inputs + submit for the e2e.

**Scope note (decided):** editable fields are firstName, lastName, nickname — the things a user actually
needs to fix. Email is read-only. No notification preferences (notifications are PM-configured rules, not
per-user prefs) — out of scope for this set.
