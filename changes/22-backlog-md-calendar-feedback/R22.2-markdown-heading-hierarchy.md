# R22.2 — Markdown heading hierarchy in preview/display

**Status:** tests passing
**Files:** `src/components/markdown-view.tsx`, `src/components/markdown-editor.tsx`

## Spec

**Problem:** `markdown-view.tsx:25` styles all headings identically — `[&_h1]:font-semibold
[&_h2]:font-semibold [&_h3]:font-semibold` with no size/weight/spacing differences — so `#`, `##`, and `###`
are indistinguishable when previewing or displaying a deliverable/subtask description. Worse, the preview
pane in `markdown-editor.tsx` renders its **own** inline `<ReactMarkdown>` (its own class list, lines 4–5),
a second render path that can drift from `MarkdownView` (used by `sortable-deliverables.tsx`).

**Approach:**

- **Consolidate:** `markdown-editor.tsx`'s preview pane renders through `MarkdownView` instead of its own
  inline `ReactMarkdown` + classes — one render path, preview always matches display.
- **Hierarchy in `MarkdownView`:** distinct arbitrary-variant styles per level, scaled for the compact
  description context (these render inside cards/rows, not as page chrome — page h1/h2 use Instrument
  Serif, but embedded markdown should stay Geist Sans and simply step down in size):
  - `h1` → `text-lg font-semibold mt-3 mb-1 first:mt-0`
  - `h2` → `text-base font-semibold mt-2.5 mb-1 first:mt-0`
  - `h3` → `text-sm font-semibold mt-2 mb-0.5 first:mt-0`
  - `h4`–`h6` → `text-sm font-medium` (rarely used; just keep them below h3)
- While in there, cover the other unstyled block elements the current class list misses so GFM previews
  read correctly: `blockquote` (left border in `--border`, pl-3, muted), `hr` (border-t my-2), and
  `pre`/`code` block background (`--secondary`, rounded, p-2, mono — inline code style already exists).
- No state, no data flow, no permissions — pure presentation in a shared client component.

**Non-obvious constraints:** `MarkdownView` is deliberately class-based (no typography plugin installed —
per the file's own comment); keep it that way rather than adding `@tailwindcss/typography`. Plain-text
descriptions must render unchanged (the component's core guarantee).
