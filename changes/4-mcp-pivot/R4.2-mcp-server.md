# R4.2 — MCP server

**Status:** done
**Files:** `src/app/api/mcp/route.ts`

## Spec

`POST /api/mcp` — implements the MCP protocol (JSON-RPC 2.0, protocol version `2024-11-05`).

**Auth:** `Authorization: Bearer <mcpToken>` → `User.mcpToken` DB lookup. 401 if token missing, invalid, or user not ACTIVE.

**Methods:** `initialize` → server capabilities · `ping` → empty · `notifications/initialized` → 204 · `tools/list` → tool list · `tools/call` → dispatch to executor.

**Tools (initial 4):**
| Name | Description | Args | RBAC |
|------|-------------|------|------|
| `list_projects` | All projects user can see | none | VIEW_ALL_PROJECTS or assignment |
| `get_project_detail` | Deliverables + open action items | `projectId` | assignment or VIEW_ALL_PROJECTS |
| `create_action_item` | Create an action item | `projectId, description, deadline?` | ASSIGN_ACTION_ITEMS or LEAD/SUBLEAD |
| `create_status_update` | Post a weekly status update | `projectId, meetingDate, plannedWork, actualProgress, blockers, nextWeekGoals` | POST_MEETING_TRACKING or SUBMIT_STATUS_UPDATES or LEAD/SUBLEAD |

All tools return JSON text in `content[0].text`.
