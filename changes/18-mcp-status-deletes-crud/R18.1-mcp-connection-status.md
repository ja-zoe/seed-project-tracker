## Spec

**Problem:** The account page's MCP section shows only a static `Token active` / `No token` pill, derived
from whether `User.mcpToken` exists. That tells the user nothing about whether an AI client is *actually*
connected, and it cannot represent an **OAuth (ChatGPT)** connection at all â€” OAuth access tokens are
Stytch-managed and leave no local record (R17.2 deferred a `McpConnection` table). So a user who connected
ChatGPT sees "No token" and assumes nothing works; a user with a stale token can't tell it's unused.

**Approach (last-used tracking â€” confirm Open Question 1):**

1. **Schema.** Add two nullable columns to `User`:
   ```prisma
   mcpTokenLastUsedAt  DateTime?
   mcpOauthLastUsedAt  DateTime?
   ```
   DDL: `ALTER TABLE "User" ADD COLUMN "mcpTokenLastUsedAt" TIMESTAMPTZ, ADD COLUMN "mcpOauthLastUsedAt" TIMESTAMPTZ;`
   Apply via `tsx scripts/apply-schema.ts`, then `prisma generate` + **restart dev** (Turbopack WASM cache).

2. **Stamp on use.** In `src/app/api/mcp/route.ts` `authenticate()`, after a successful resolution, set the
   matching column to `now()`:
   - static-token branch (non-JWT, matched `User.mcpToken`) â†’ `mcpTokenLastUsedAt`.
   - OAuth branch (`verifyOAuthAccessToken` â†’ resolved SEED user) â†’ `mcpOauthLastUsedAt`.
   Do it as a non-blocking best-effort write wrapped in `try/catch` (a stamp failure must never 500 a tool
   call). To avoid a write on every single request, only update when the stored value is null or older than
   ~5 minutes (cheap throttle: compare in-memory after the lookup we already do, or just always write â€”
   volume is low; default to the throttle to keep DB writes down). For the OAuth branch the stamp belongs
   in `verifyOAuthAccessToken`'s success path (it already loads the user) or right after it returns in
   `authenticate()`; put it in `authenticate()` so both branches stamp in one place.

3. **Read on the account page.** `src/app/(app)/account/page.tsx` already loads the current user; extend the
   `select` to include `mcpToken` (presence only), `mcpTokenLastUsedAt`, `mcpOauthLastUsedAt`. Pass
   `hasToken`, `tokenLastUsed`, `oauthLastUsed` (ISO strings or null) to `MpcTokenSection`.

4. **Render status.** In `mcp-token-section.tsx`, replace the single status pill with a small two-row
   "Connections" block (keep the existing generate/regenerate/revoke controls below it):
   - **Personal access token:** if no token â†’ muted "No token". If token but `tokenLastUsed` null â†’
     "Token active Â· never used". Else "Token active Â· last used {relative}". Green dot when last used â‰¤ 7d.
   - **OAuth (ChatGPT & web hosts):** if `oauthLastUsed` null â†’ muted "Not connected". Else
     "Connected via OAuth Â· last used {relative}". Green dot when â‰¤ 7d.
   Copy must say "last used" (not "currently connected") â€” we can't detect remote revocation, only activity.

5. **Revoke clears the stamp.** `revokeMcpToken()` in `src/lib/actions/account.ts` should also null
   `mcpTokenLastUsedAt` so the status doesn't show a stale "last used" after the token is gone.

**State:** all server-side (DB timestamps) â†’ page server component â†’ client section props. No new client
state beyond display. **Relative-time:** check `src/lib/utils.ts` for an existing formatter; if none, add a
small `formatRelative(date)` ("just now", "3h ago", "5d ago").

**Non-obvious constraints:** the OAuth stamp must not depend on a `McpConnection` table (intentionally not
built). The throttle keeps `authenticate()` from hammering the DB on multi-call tool sessions.
