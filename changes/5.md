### R5.1 — Notification modal positioning fix

**File:** `src/components/notification-bell.tsx`, line 94

**Root cause:** The popup is `absolute bottom-full right-0`. The bell lives at the bottom-right corner of the left sidebar. `right-0` aligns the popup's right edge with the bell button's right edge (~212 px from the left edge of the viewport). Because the popup is `w-80` (320 px), it extends 320 px **leftward** from that anchor — going ~108 px off-screen to the left.

**Fix:** Change `right-0` → `left-0`. This aligns the popup's **left** edge with the bell container's left edge and extends it 320 px **rightward** into the main content area, where it's fully visible.

```diff
- className="absolute bottom-full right-0 mb-2 w-80 ..."
+ className="absolute bottom-full left-0 mb-2 w-80 ..."
```

No other positioning changes needed.

### R5.2 — MCP: deliverable & subtask CRUD

**File:** `src/app/api/mcp/route.ts`

Add 6 tools to the `TOOLS` array and handle them in `executeTool`. RBAC mirrors the existing server actions in `src/lib/actions/deliverables.ts`.

#### Tool definitions

| Name | Required args | Optional args | RBAC |
|------|--------------|---------------|------|
| `create_deliverable` | `projectId`, `title`, `targetDate` (ISO date) | `description`, `startDate`, `group` | MANAGE_MILESTONES **or** project LEAD/SUBLEAD |
| `update_deliverable` | `deliverableId` | `title`, `targetDate`, `description`, `startDate`, `status` (TimelineStatus), `group` | MANAGE_MILESTONES **or** project LEAD/SUBLEAD |
| `delete_deliverable` | `deliverableId` | — | MANAGE_MILESTONES only (destructive) |
| `create_subtask` | `deliverableId`, `title` | `description`, `dueDate`, `assigneeId` | any project member **or** MANAGE_MILESTONES |
| `update_subtask` | `subtaskId` | `title`, `description`, `dueDate`, `assigneeId`, `status` (TimelineStatus) | any project member **or** MANAGE_MILESTONES |
| `delete_subtask` | `subtaskId` | — | MANAGE_MILESTONES **or** project LEAD/SUBLEAD |

#### Implementation notes

- `create_deliverable`: look up project membership for LEAD/SUBLEAD check. `orderIndex` = count of existing deliverables. Set `completed: false` by default.
- `update_deliverable`: fetch `deliverable.projectId` first to check membership. When `status === "COMPLETE"`, set `completed: true` and `completedDate: new Date()`.
- `delete_deliverable`: no membership fallback — MANAGE_MILESTONES is required. Cascade deletes subtasks (FK onDelete: Cascade).
- `create_subtask`: fetch `deliverable.projectId` to check membership. `orderIndex` = count of existing subtasks on that deliverable.
- `update_subtask`: fetch subtask + `deliverable.projectId` for membership check. When `status === "COMPLETE"`, set `completedAt: new Date()`; on any other status set `completedAt: null`.
- `delete_subtask`: fetch subtask + deliverable for LEAD/SUBLEAD check; fallback to MANAGE_MILESTONES if no membership.

All tools return `{ updated: { id, ...fields } }` or `{ deleted: { id } }` or `{ created: { id, ...fields } }` in `content[0].text`.

### R5.3 — MCP: member listing, action item management, personal task view

**File:** `src/app/api/mcp/route.ts` (continued)

Add 4 more tools.

#### `list_members`

Lists users with optional filtering. Mirrors a REST-style query endpoint.

**Args (all optional):**
- `projectId` — restrict to members assigned to this project
- `projectRole` — `"LEAD" | "SUBLEAD" | "MEMBER"` — only valid combined with `projectId`
- `roleName` — filter by global Role name (e.g., `"Project Manager"` for eboard members)
- `status` — `"ACTIVE" | "PENDING" | "SUSPENDED"` (default: `"ACTIVE"`)

**RBAC:**
- With `VIEW_ALL_PROJECTS`: returns all matching users across the org.
- Without: if `projectId` provided and the caller is a member, returns that project's members. Otherwise returns only the caller.

**Return:** `{ members: [{ id, name, email, status, roleName, projectRole? }] }`

Implementation — query `User` with `include: { role: { select: { name: true } } }`. When `projectId` is present, join through `ProjectAssignment`. Apply `where` clauses for `status` and role filters. If `roleName` is provided, filter via `role: { name: roleName }`.

#### `update_action_item`

Allows closing, reopening, or editing an action item.

**Args:** `actionItemId` (required), `status?: "OPEN" | "DONE"`, `deadline?: string`, `description?: string`

**RBAC:** CLOSE_ACTION_ITEMS **or** ASSIGN_ACTION_ITEMS **or** project LEAD/SUBLEAD. Fetch `actionItem.projectId` to check membership.

Sets `completedAt: new Date()` when `status` is set to `"DONE"`, clears it on `"OPEN"`.

**Return:** `{ updated: { id, status, deadline, description } }`

#### `list_action_items`

Query action items with filters.

**Args (all optional):** `projectId`, `status` (`"OPEN" | "DONE"`), `assignedToMe` (bool — filters to `ownerId === caller.id`)

**RBAC:** VIEW_ALL_PROJECTS to query any project; otherwise limited to assigned projects.

**Return:** `{ actionItems: [{ id, description, status, deadline, projectId, projectName, ownerName }] }` — up to 50 results, ordered by deadline asc (nulls last).

#### `get_my_subtasks`

Returns all subtasks assigned to the calling user.

**Args (optional):** `status` (`"NOT_STARTED" | "IN_PROGRESS" | "BLOCKED" | "COMPLETE"`)

No special RBAC beyond authentication — always scoped to caller.

**Return:** `{ subtasks: [{ id, title, status, dueDate, deliverableId, deliverableTitle, projectId, projectName }] }` — ordered by dueDate asc (nulls last), up to 50 results.

### R5.4 — Logo + favicon follow-up fixes

Playwright screenshots revealed three bugs with R5.0:

**White box around logo** — The PNG has white pixels in background areas. Against the warm `#F4F1EA` page canvas visible through any layout gap, this white box is noticeable. Fix: add `style={{ mixBlendMode: 'multiply' }}` to both `<Image>` usages. Multiply makes white pixels absorb the background color (effectively transparent) while preserving dark logo content.

**Logo too small** — 22×22px is barely visible. Increase to `width={36} height={36}` on both sidebar instances. The sidebar content area is 184px wide (224 − 40px padding); "SEED Tracker" text is ~100px, so 36 + 8(gap) + 100 = 144px — fits comfortably.

**Favicon not applying** — `src/app/favicon.ico` takes precedence over `metadata.icons`. Fix: delete `favicon.ico` and copy `public/seed-logo-transparent.png` to `src/app/icon.png`. Next.js App Router auto-detects `app/icon.{png,jpg,ico}` and serves it as `<link rel="icon">`. Remove the now-redundant `metadata.icons` line from `layout.tsx`.

**Dev-login page** — Still uses `<Leaf>` icon from Phosphor. Replace with `<Image>` (same approach as sidebar, `width={24} height={24}` with `mix-blend-mode: multiply`).

**Files:** `src/components/sidebar.tsx`, `src/app/dev-login/page.tsx`, `src/app/layout.tsx`, `src/app/icon.png` (new), `src/app/favicon.ico` (delete)
