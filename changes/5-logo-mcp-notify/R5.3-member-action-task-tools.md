# R5.3 — MCP: member listing, action item management, personal task view

**Status:** done
**Files:** `src/app/api/mcp/route.ts` (shared with R5.2 — sequence, don't parallelize)

## Spec

Add 4 more tools to the MCP server.

### `list_members`
Lists users with optional filtering.

**Args (all optional):**
- `projectId` — restrict to members assigned to this project
- `projectRole` — `"LEAD" | "SUBLEAD" | "MEMBER"` (only valid with `projectId`)
- `roleName` — filter by global Role name (e.g. `"Project Manager"` for eboard)
- `status` — `"ACTIVE" | "PENDING" | "SUSPENDED"` (default `"ACTIVE"`)

**RBAC:** with `VIEW_ALL_PROJECTS`, returns all matching users org-wide. Without: if `projectId` is given and caller is a member, returns that project's members; otherwise returns only the caller.

**Return:** `{ members: [{ id, name, email, status, roleName, projectRole? }] }`. Query `User` with `include: { role: { select: { name: true } } }`; when `projectId` present, join through `ProjectAssignment`; apply `where` for status/role filters; `roleName` → `role: { name: roleName }`.

### `update_action_item`
Close, reopen, or edit an action item.

**Args:** `actionItemId` (required), `status?: "OPEN" | "DONE"`, `deadline?: string`, `description?: string`
**RBAC:** CLOSE_ACTION_ITEMS **or** ASSIGN_ACTION_ITEMS **or** project LEAD/SUBLEAD (fetch `actionItem.projectId` for membership).
Sets `completedAt: new Date()` on `"DONE"`, clears it on `"OPEN"`.
**Return:** `{ updated: { id, status, deadline, description } }`

### `list_action_items`
**Args (all optional):** `projectId`, `status` (`"OPEN" | "DONE"`), `assignedToMe` (bool → `ownerId === caller.id`)
**RBAC:** VIEW_ALL_PROJECTS to query any project; otherwise limited to assigned projects.
**Return:** `{ actionItems: [{ id, description, status, deadline, projectId, projectName, ownerName }] }` — up to 50, deadline asc (nulls last).

### `get_my_subtasks`
**Args (optional):** `status` (`"NOT_STARTED" | "IN_PROGRESS" | "BLOCKED" | "COMPLETE"`)
**RBAC:** authentication only — always scoped to caller.
**Return:** `{ subtasks: [{ id, title, status, dueDate, deliverableId, deliverableTitle, projectId, projectName }] }` — up to 50, dueDate asc (nulls last).
