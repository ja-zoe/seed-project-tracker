## Spec

**Problem:** The brand mark was the generic Phosphor `Plant` icon. Replace it with the real `seed-logo-transparent.png` across the sidebar (mobile drawer header + desktop header), the dev-login page, and the favicon.

**Approach:**

Replace both sidebar logo occurrences with a Next.js `<Image>`:

```tsx
import Image from "next/image";

// Replace both:
<Plant size={18} weight="fill" className="text-primary" />
// With:
<Image src="/seed-logo-transparent.png" alt="SEED" width={24} height={24} className="object-contain" />
```

Remove `Plant` from the `@phosphor-icons/react` import (verify other icon imports — `Folders`, `CheckSquare`, etc. — are unaffected). Dev-login page uses `<Leaf>` for the same purpose — replace with the same `<Image>` approach.

### R5.4 — follow-up fixes (Playwright screenshots)

Three bugs surfaced after R5.0 shipped:

- **White box around logo.** Root cause: Next.js Image optimization was converting the PNG to WebP (q=75), flattening the alpha channel. Fix: add the `unoptimized` prop to all seed-logo `<Image>` usages — the PNG is then served directly from `/public/`, preserving RGBA transparency. (An earlier `mixBlendMode: 'multiply'` attempt was superseded by `unoptimized`.)
- **Logo too small.** 22×22 was barely visible. Increased to `width={36} height={36}` on both sidebar instances. Sidebar content area is 184px (224 − 40 padding); "SEED Tracker" text ~100px, so 36 + 8 gap + 100 = 144px — fits.
- **Favicon not applying.** `src/app/favicon.ico` takes precedence over `metadata.icons`. Fix: delete `favicon.ico`, copy the logo to `src/app/icon.png` (App Router auto-detects `app/icon.png` as the favicon), and remove the redundant `metadata.icons` line from `layout.tsx`.
- **Dev-login page** `<Leaf>` replaced with the seed-logo `<Image>` (same approach, `unoptimized`).

- 2026-06-27 — R5.4 implemented and verified via Playwright: transparent logo, favicon link pointing to `icon.png`.
