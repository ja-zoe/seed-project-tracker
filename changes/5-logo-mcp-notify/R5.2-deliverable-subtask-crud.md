# R5.2 — MCP: deliverable & subtask CRUD

**Status:** done
**Files:** `src/app/api/mcp/route.ts` (shared with R5.3 — sequence, don't parallelize)

## Spec

**Problem:** The MCP server only exposed read + action-item/status tools. Add full deliverable and subtask CRUD so a connected AI client can manage the project plan.

**Approach:** Add 6 tools to the `TOOLS` array and handle them in `executeTool`. RBAC mirrors the existing server actions in `src/lib/actions/deliverables.ts`.

| Name | Required args | Optional args | RBAC |
|------|--------------|---------------|------|
| `create_deliverable` | `projectId`, `title`, `targetDate` (ISO) | `description`, `startDate`, `group` | MANAGE_MILESTONES **or** project LEAD/SUBLEAD |
| `update_deliverable` | `deliverableId` | `title`, `targetDate`, `description`, `startDate`, `status` (TimelineStatus), `group` | MANAGE_MILESTONES **or** LEAD/SUBLEAD |
| `delete_deliverable` | `deliverableId` | — | MANAGE_MILESTONES only (destructive) |
| `create_subtask` | `deliverableId`, `title` | `description`, `dueDate`, `assigneeId` | any project member **or** MANAGE_MILESTONES |
| `update_subtask` | `subtaskId` | `title`, `description`, `dueDate`, `assigneeId`, `status` (TimelineStatus) | any project member **or** MANAGE_MILESTONES |
| `delete_subtask` | `subtaskId` | — | MANAGE_MILESTONES **or** LEAD/SUBLEAD |

Implementation notes:
- `create_deliverable`: look up project membership for the LEAD/SUBLEAD check. `orderIndex` = count of existing deliverables. `completed: false` by default.
- `update_deliverable`: fetch `deliverable.projectId` first for membership. When `status === "COMPLETE"`, set `completed: true`, `completedDate: new Date()`.
- `delete_deliverable`: no membership fallback — MANAGE_MILESTONES required. Cascade deletes subtasks (FK onDelete: Cascade).
- `create_subtask`: fetch `deliverable.projectId` for membership. `orderIndex` = count of existing subtasks on that deliverable.
- `update_subtask`: fetch subtask + `deliverable.projectId` for membership. `status === "COMPLETE"` → `completedAt: new Date()`; any other status → `completedAt: null`.
- `delete_subtask`: fetch subtask + deliverable for LEAD/SUBLEAD check; fallback to MANAGE_MILESTONES if no membership.

All tools return `{ created | updated | deleted: { id, ...fields } }` in `content[0].text`.
